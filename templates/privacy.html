KpopWave Daily Privacy Policy

プライバシーポリシー

最終更新日: 2026年5月16日  |  Privacy Policy (Last updated: May 16, 2026)

本ポリシーは、KpopWave Daily（以下「本サービス」）が Meta Platforms の Threads API を使用して K-POP 女性アイドルに関するニュースを自動投稿するにあたり、取り扱う情報と利用目的を説明するものです。

1. サービス概要

本サービスは、公開されている K-POP ニュースサイトの RSS フィードから記事を収集し、 要約した内容を Threads アカウント（@kpopwave.daily）へ自動投稿する 個人運営のツールです。一般ユーザーのアカウントデータを収集・処理する機能はありません。

2. 収集する情報

本サービスが取り扱う情報は、運営者自身の Threads アカウントに関するものに限られます。

情報の種類内容取得方法
Threads ユーザー ID 投稿先アカウントの識別子 Meta OAuth 認証フローにより取得
Threads アクセストークン API 呼び出しに使用する認証情報 Meta OAuth 認証フローにより取得
RSS 記事データ 公開ニュースサイトの記事タイトル・本文・URL 公開 RSS フィードから自動取得

一般ユーザーの個人情報は一切収集しません。 本サービスは単一の Threads アカウント（運営者のもの）のみを管理します。

3. 情報の利用目的

  • Threads API を通じた自動投稿の実行
  • アクセストークンの有効性確認および更新
  • 投稿履歴・ステータスの管理（ローカルデータベースへの記録）

収集した情報を上記以外の目的に使用することはありません。

4. 第三者への情報提供

本サービスは、取得した情報を第三者（広告事業者・データブローカー等）に販売・提供・貸与しません。 情報は運営者のローカル環境にのみ保存され、外部サーバーへ転送されません。

ただし、投稿処理のために以下のサービスへ情報が送信されます。

  • Meta Platforms（Threads API） — 投稿内容およびアクセストークン
  • Anthropic（Claude API） — 要約生成のための記事テキスト（個人情報を含まない公開情報のみ）

5. データの保存・削除

収集したデータは運営者のローカル環境（SQLite データベース）に保存されます。 アクセストークンは Meta のポリシーに従い最長 60 日で失効します。

データの削除を希望する場合、または本サービスによる Threads アカウントとの連携解除を希望する場合は、 下記の連絡先までご連絡ください。速やかに対応いたします。

また、Threads アプリの設定画面からもアプリ連携を直接解除できます：
Threads アプリ → プロフィール → 設定 → アカウント → アプリとウェブサイト

6. Meta Platform ポリシーへの準拠

本サービスは以下の Meta ポリシーに準拠して運営されます。

  • Meta Platform Terms
  • Meta Developer Policies
  • Threads Supplemental Terms

本サービスが使用する Threads API の権限は以下のとおりです。

  • threads_basic — プロフィール情報の読み取り
  • threads_content_publish — テキスト投稿の公開

7. Cookie・トラッキングについて

本サービスはアナリティクスや広告目的の Cookie を使用しません。 OAuth 認証フロー中にセッション管理のための一時的な Cookie を使用しますが、 認証完了後には削除されます。

8. ポリシーの変更

本ポリシーは必要に応じて更新することがあります。 重要な変更がある場合は、このページの「最終更新日」を変更してお知らせします。

9. お問い合わせ・データ削除リクエスト

本ポリシーに関するお問い合わせ、またはデータ削除のご要望は以下までご連絡ください。

Threads: @kpopwave.daily
Email: mktiskw1@gmail.com

ご連絡から 72 時間以内に対応いたします。

Summary in English (for Meta Platform Review)

KpopWave Daily is a personal, single-account automated posting tool that collects publicly available K-POP news via RSS feeds, summarizes them using the Claude API (Anthropic), and publishes the summaries to a single Threads account (@kpopwave.daily) operated by the developer.

Data collected: Threads User ID and Access Token of the operator's own account only. No end-user data is collected.

Data use: Solely for authenticating and publishing posts to the operator's Threads account via the Threads API.

Data sharing: No user data is sold, shared, or transferred to any third party. Data is stored locally on the operator's machine.

Data deletion: Users can disconnect the app from within the Threads app settings at any time. Deletion requests can also be sent to mktiskw1@gmail.com.

Permissions used: threads_basic, threads_content_publish
